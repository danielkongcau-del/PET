# 首个垂直切片：验收清单

状态：需求已冻结，供开发与验收共同使用。目标平台仅为 Windows 11，首阶段只要求开发模式运行。

## 一句话目标

用一条开发命令启动透明桌面猫与本地 Python 生成器；猫的短时运动和姿态每次在线生成，可以在窗口顶边行走、自主跳到附近窗口顶边，并在实体像素被点击时实时反应，同时由桌面宿主保证窗口碰撞、可见性和故障安全。

## 自动化门禁

以下命令均须返回 0：

```powershell
conda activate pet-core
python environment/smoke_test.py
python -m unittest discover -s tests/protocol -v
python -m unittest discover -s tests/integration -v
```

协议门禁检查 JSON Schema 与 Python 镜像共同接受固定 fixture，并共同拒绝未知字段、错误版本、非有限数值和非法轨迹时间。生成器集成测试必须真实启动 `services/generator/run.py`，完成 `hello/ready`、`ping/pong`、普通状态规划、点击反应、取消后存活；不允许用 mock 生成器代替。

若桌面端提供其自己的测试命令，则启动总命令执行前也必须通过。测试和调试输出不能写入协议 stdout。

## 手工功能验收

### 启动与角色

- [ ] 仓库根目录的一条开发命令同时启动 Electron 宿主和 `pet-core` 中的 Python 生成器。
- [ ] 生成器在 8 秒内发送合法 `ready`；未 ready 前宿主不执行模型轨迹。
- [ ] 猫以 `48×48` 内部像素画、最近邻放大为 `96×96` 显示；白色身体、黑色轮廓、尖耳和卷尾与参考图保持可辨识一致。
- [ ] 猫允许拆分头、身体、四肢和尾巴并实时形变；正常交互不播放预录动作片段。
- [ ] 窗口透明、无边框、普通桌面环境中置顶，不出现白底或插值模糊。

### 顶边行走

- [ ] 猫只把窗口顶部的水平可见区段视作可行走表面，不在左右边攀爬或底边倒挂。
- [ ] 猫停留和行走时脚底锚点落在表面上；不能穿过窗口、瞬移或超出当前显示器工作区。
- [ ] 没有合适窗口时，猫落到当前显示器工作区底边；不得覆盖任务栏。
- [ ] 至少连续观察 2 分钟，可看到在线生成的速度、身体倾斜、压缩和起伏存在自然变化，而不是固定周期循环。

### 自主跳跃

- [ ] 猫自主从当前显示器的可见、可达窗口中选择附近目标，不要求鼠标指定目标。
- [ ] 起跳、空中轨迹和落地由当前 `horizon_plan` 连续驱动，最终落点由宿主安全层限制在目标表面。
- [ ] 移动目标窗口时 350 ms 内取消或重新规划；最小化、关闭目标窗口后，猫安全跳往其他表面或落到工作区底边。
- [ ] 首阶段不跨显示器跳跃，但每个显示器及不同 DPI 缩放下的坐标与碰撞正确。

### 点击反应

- [ ] 左键点击猫的非透明像素时，点击由猫拦截，不传给下方窗口。
- [ ] 同一 `96×96` 窗口的透明像素区域将点击传给下方程序。
- [ ] 点击后可见反应的 p95 延迟低于 100 ms；反应为在线生成的压扁回弹、受惊后退、短跳或转身等变化，首阶段无声音。
- [ ] 点击事件只消费一次；状态刷新不能重复触发同一 click id。

### 系统可见性

- [ ] 全屏游戏或全屏视频激活时猫自动隐藏，退出全屏后恢复到安全位置。
- [ ] 猫不覆盖任务栏、开始菜单和系统安全界面；宿主不能试图绕过 Windows 安全桌面。
- [ ] 托盘菜单具备暂停/恢复、调试层开关、重启生成器和退出；首阶段不需要完整设置页。

### 调试层

- [ ] 默认关闭；开启后显示候选/有效表面、脚底锚点、目标、短时轨迹、当前行为、plan id、`based_on_seq`、生成延迟和安全层修正。
- [ ] 调试层本身不拦截面向下方应用的点击，也不改变模型输入或随机种子。

## 故障与安全验收

- [ ] 杀死 Python 进程后 250 ms 内停止执行未完成 plan，并切换为宿主的安全待机；猫不会冻结在半空、消失到屏幕外或继续陈旧跳跃。
- [ ] 宿主自动尝试重启生成器；新会话必须重新 `hello/ready`，不能复用旧 session 的 plan。
- [ ] 宿主拒绝过期 plan、未来 `based_on_seq`、过旧状态、已取消 plan、NaN/Infinity、非法点间隔、失效 surface 和越界姿态。
- [ ] 冲突优先级为：用户拖拽 > 窗口拓扑变化 > 安全层 > 生成 plan > fallback。首版虽无拖拽功能，仍保留最高优先级取消入口。
- [ ] 状态输入采用容量为 1 的 latest-state 队列；生成器繁忙时丢弃旧状态，不积压追赶。
- [ ] 连续运行 30 分钟无持续内存/句柄增长、无僵尸 Python 进程、无 stdout 协议污染。

## 性能目标

- [ ] 显示和宿主执行目标为 60 FPS；world state 目标为 20 Hz；短轨迹采样目标为约 30 Hz，重新规划目标为 10–15 Hz。
- [ ] 首版随机自回归生成器的 plan 响应 p95 小于 100 ms；记录 `plan_latency_ms`、队列深度、拒绝/修正/重启计数。
- [ ] 日常常驻显存尽量低于 1 GB；开发、模型训练和性能诊断阶段可以临时超过。

## 隐私与日志

- [ ] 全部处理只在本机完成；默认不联网、无遥测。
- [ ] 首版实际输入仅使用鼠标、点击和窗口几何；截图与键盘接口可以预留但默认关闭。
- [ ] 截图和原始按键不落盘；真实桌面交互录制默认关闭，只有用户明确开启后才采集。
- [ ] 窗口标题和程序名只允许宿主用于系统窗口过滤，不交给动作模型、不写入日志。
- [ ] 日志仅含时间、匿名窗口矩形、行为、轨迹和性能指标，并滚动覆盖避免无限增长。

## 明确不在首阶段

- 拖拽猫、声音、完整设置界面、安装包、开机自启；
- 左右窗口边攀爬、底边倒挂、跨显示器跳跃；
- 根据屏幕内容或应用语义决定行为；
- 神经网络逐像素生成完整 RGBA 帧；
- 真实桌面交互数据采集和扩散模型训练。

首阶段先以随机自回归轨迹生成器打通完整链路。后续以程序化桌面模拟器训练条件扩散动作模型时，必须保持 v1 的宿主安全边界；若线格式需要扩展则明确升级协议版本，不在 v1 中静默增加字段。
